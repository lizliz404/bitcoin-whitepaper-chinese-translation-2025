### 为什么在 2025 年重新翻译它？

**起初只是因为好奇心。**

在研究 Web3 和区块链时，我反复看李笑来老师的文章。他好像总在若有若无地告诉你，阅读白皮书是少数人会做的、正确而困难的事：

> [“如果你有兴趣耐心反复阅读比特币白皮书，《比特币：一种点对点的电子现金系统》，那么你就会发现……”](http://lixiaolai.com/#/bitcoin-whitepaper-cn-en-translation/Bitcoin-Whitepaper-EN-CN.html)

他指出，大多数人只是“失败的投机者”，因为他们甚至不知道自己在交易什么：

> [“……究竟有几个人真正读过比特币的白皮书呢？……他们甚至实际上并不知道自己手中正在买卖的究竟是什么……玩命地把自己变成一个有知者 —— 这才是明智的选择，不仅明智，这才是强者的选择。”](http://lixiaolai.com/#/the-self-cultivation-of-leeks/cn)

最后，他留下一句貌似无奈的感慨：

> [“……这东西真没几个人认真看的…… 十年过来，一如既往地如此。”](https://github.com/xiaolai/bitcoin-whitepaper-chinese-translation)

这激起了我的挑战欲。而事实证明，“翻译”是最高效的精读方式——它强迫你直面源头，无法自欺欺人地跳过任何一个晦涩的细节。这场“硬核”的阅读练习，帮我建立起了面对复杂概念的认知底气。

**白皮书的三重“干净”**

在逐句翻译的过程中，我真切感受到了这份诞生于 2008 年的文档的力量，我称之为三重“干净”：

1. **知识的干净**：信息密度极高，没有废话，让你直接越过无数“二道贩子”加工后的信息噪音。
2. **氛围的干净**：纯粹的技术探讨，只讲“存在什么问题”以及“我的系统怎么解决”，没有市场的 FOMO 与焦虑。
3. **立场的干净**：它只是一个解决双花问题的电子现金系统方案，不夹带私货，不鼓吹意识形态。

在这个信息爆炸的时代，回归这样一份干净的文本，本身就是一种极好的思维清理。

### 开发后记：一场与“完美主义”的荒诞搏斗

> **写给后来的 Forker 和我自己的一段复盘。**

做这个项目，我前后花了一个月的时间。听起来很感人是？但说出来不怕你笑话，**翻译那份核心的 `zh.md` 文档，我其实只花了一周。**

剩下的三周，**我在钻牛角尖。**

为了让我的在线版网页样式，能和李笑来老师那一版“一模一样”，我一头扎进了前端样式的泥潭里。我像个考古学家一样去扒那个网页的 HTML 和 CSS，试图把每一个细节、每一处边距都像素级地复刻过来。

结果直到最后才发现这是手段和目的间的混淆：那一大坨复杂得让人头秃的代码，只是他用 Typora 软件导出的一份“死”的 HTML 静态文件而已！

那一刻我真的觉得自己蠢爆了。我花了将近一个月的时间，去跟一个软件自动生成的僵硬代码死磕。

如果我早点退一步看问题：
- 如果我只要那个效果，我完全可以自己下个 Typora，把翻译好的 MD 丢进去直接导出一份。
- 如果我想自己搭建，用最简单的 `fetch` 获取 MD 文件，套一个现成的 Markdown 渲染库（比如 marked.js），再加上简单的 CSS，几小时就能搞定。
- 退一万步说，GitHub 原生的 Markdown 渲染不就已经足够好看了吗？读者完全可以直接看仓库里的 `zh.md` 啊！

**技术只是手段，内容才是核心。** 

我把 80% 的时间，花在了解决一个完全偏离核心价值（即翻译文本本身）的“伪需求”上，这大概就是完美主义发作时最真实的丑态。

把这段黑历史写在这里，是想提醒自己，也提醒看到这里的你：
**Done is better than perfect.** 别让极客的执念和无谓的完美主义，绑架了真正该做的事。
